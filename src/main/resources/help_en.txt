[Parameter Specification]

<b>--url</>
    Specify the API URL (default is "https://api.openai.com/v1/chat/completions")

<b>--model</>
    Specify the gpt model (default is "gpt-3.5-turbo")

<b>--apikey</>
    Specify the API key (if not specified, it will be fetched from the environment variable "OPENAI_API_KEY")

<b>--resource</>
    Specify the type of input resource (default is "others")
    Available resource types are:
    <b>java</> : Java source code (targeting files with ".java" extension)
    <b>js</> : JavaScript source code (targeting files with ".js" extension)
    <b>python</> : Python source code (targeting files with ".py" extension)
    <b>sql</> : SQL code (targeting files with ".sql" extension)
    <b>page</> : Page files (targeting ".page", ".html", ".htm", ".xhtml", ".jsp" extensions)
    <b>shell</> : Shell scripts (targeting ".sh", ".bash", ".ksh", ".bash" extensions)
    <b>others</> : Generic resources other than the above

<b>--output</>
    Specify what processing should be performed on the resource and what you want to output (default is "others")
    Available output types are:
    <b>spec</> : Specification document
    <b>summary</> : Summary
    <b>review</> : Review results & improvement suggestions
    <b>api</> : API list (only when resource is java)
    <b>const</> : List of constants (only when resource is java)
    <b>message</> : List of messages (only when resource is java)
    <b>exception</> : List of exceptions (only when resource is java)
    <b>validation</> : List of validations (only when resource is java)
    <b>er</> : E-R diagram (only when resource is sql)
    <b>others</> : Generic outputs other than the above

<b>--output-scale</>
    Specify the approximate size of the output (default is "nolimit")
    <b>small</> : Small size (about 50 characters)
    <b>medium</> : Medium size (about 200 characters)
    <b>large</> : Large size (about 500 characters)
    <b>nolimit</> : No limit

<b>--src</>
    Specify the file or directory of the source to be read (mandatory specification)
    When specifying a directory, files are processed recursively
    When specifying a ZIP file, the contents are also processed recursively

<b>--regex</>
    Specify a regular expression string to filter the sources to be read (optional)

<b>--dest</>
    Specify the directory name or filename of the output markdown file in full path
    If a directory name is specified, the default filename becomes "brain2doc-{resource name}-{output type}-{yMMddHHmmss}.md"
    (Example: "brain2doc-java-review-20230924104914.md")
    If this option isn't specified, it will be output in the same directory as the source file with the default filename

<b>--lang</>
    Specify the input/output language (default is Japanese)
    <b>ja</> : Japanese
    <b>en</> : English

<b>--template</>
    Specify any template file in full path (optional)
    Only differences from the default template file will be overwritten

<b>--proxyURL</>
    Specify the proxy server URL when making API calls

<b>--connect-timeout</>
    Specify the connection timeout duration in seconds for API calls (default is 10 seconds)

<b>--timeout</>
    Specify the reading timeout duration in seconds for API calls (default is 300 seconds)

<b>--retry-count</>
    Specify the maximum number of retries for API calls when a reading timeout occurs (default is 3 times)

<b>--retry-interval</>
    Specify the retry interval in seconds when a reading timeout occurs for API calls (default is 60 seconds)

<b>--auto-split</>
    If this option is specified, if a single API call reaches the token limit, it will automatically switch to a mode that splits and resends it to the appropriate size
    If not specified, the corresponding source will be skipped
